# Windows 介绍

## MicroSoft

## MS-DOS

微软（Microsoft）由比尔·盖茨与保罗·艾伦于 1975年 创立。最初，他们并不生产“OS”，而是为微型计算机提供 BASIC 解释器。直到 1981 年，由于 IBM 需要为其个人电脑寻找操作系统，微软通过收购并授权的方式推出了 MS-DOS。

那是一个只有黑底白字、通过命令行交互的时代。MS-DOS 虽然简陋，却为微软赚到了第一桶金，并奠定了微软在 PC 市场的霸权。

## 发展过程

MS-DOS (1981)
Windows 最初并不是一个完整的操作系统，而是运行在 MS-DOS 之上的图形界面扩展。

Windows 1.0（1985）
提供窗口、多任务（非常有限）、鼠标操作，但功能简单，影响力有限。

Windows 2.0（1987）
支持窗口重叠，开始出现更实用的软件生态（如 Excel）。

Windows 3.0（1990–1992）
Windows 3.1
真正流行起来，图形界面更成熟，开始成为主流 PC 平台。

这一阶段本质：图形界面 + DOS 内核。

Windows 95
引入了 Win32 API 和 注册表，彻底淘汰了旧的配置文件。引入开始菜单、任务栏、桌面模型（沿用至今）。

Windows 98
改进硬件支持，增强稳定性。

Windows Me（2000）
过渡产品，口碑较差。

Windows 3.1 （1993）
全新架构，强调稳定性、安全性。这是 Windows 真正强大的开始。由 DEC 传奇工程师 Dave Cutler 领导开发，具备 32 位抢占式多任务能力，不再依赖 DOS。

Windows NT 4.0
将 Win95 的界面带入了稳定的 NT 内核。

Windows 2000 (NT 5.0)
这是 NT 路线的集大成者，引入了 Active Directory (活动目录)，成为全球企业 IT 架构的骨干。

Windows Server 2003
强调安全性的转折点，是历史上最长寿的服务器系统之一。

Windows XP（2001）
首次统一消费级与企业级，基于 NT 内核，稳定性大幅提升，成为经典。代号 Whistler。微软终于完成了“内核归一”，将脆弱的 9x 路线彻底合并到强大的 NT 路线。

Windows Vista（2007）
引入 Aero 特效、安全机制（UAC）。代号 Longhorn。这是一场宏大却失败的技术演进，试图重写驱动模型（WDDM）和文件系统（WinFS），最终因步子迈得太大导致性能雪崩。

Windows 7 (2009)
Vista 的“完成版”，也是 x64 架构真正普及的起点。

Windows 8 / 8.1 (2012)：为了对抗 iPad，微软强推 WinRT 架构和 Metro 界面，大幅改版，强调触控，“Metro UI”，取消开始菜单，引发争议,试图在 ARM 硬件上运行 Windows。虽然失败，但为后来的 Microsoft Store 铺了路。

Windows 8.1（2013）
回调部分传统设计，缓解用户不满。

Windows 10 (2015)
微软宣称这是“最后一个版本的 Windows”，提出“Windows 即服务”（持续更新而非大版本）。。它引入了 WSL (Windows Subsystem for Linux)。
恢复开始菜单（融合 Win7 + Win8），引入虚拟桌面、通知中心，推动 Universal Windows Platform（但生态有限），深度整合云（Microsoft Account、OneDrive）

Windows 11 (2021)
强制要求 TPM 2.0 安全模块，彻底放弃了对部分旧硬件的支持，UI 转向圆角浮窗风格。

Windows 11 2024/2025 更新
深度集成 AI 能力，并在 Snapdragon X Elite 等 ARM 芯片上实现了堪比 x86 的运行效率。


## Windows 与 macOS 的起源

1985 年，Windows 1.0 诞生。它本质上只是运行在 MS-DOS 之上的一个图形化壳子。

这里有一段绕不开的公案：Windows 与 macOS 的起源。
当时史蒂夫·乔布斯指责盖茨“抄袭”了 Macintosh 的图形界面，而盖茨那句著名的回应至今仍被津津乐道：

“我们都有个有钱的邻居叫施乐（Xerox），我闯进他家想偷电视机，却发现你已经把它偷走了。”

事实上，两家都从施乐帕罗奥多研究中心（Xerox PARC）汲取了灵感。Windows 的早期版本（1.0 到 3.1）表现平平，直到 Windows 95 的出现，才真正开启了图形操作系统的现代篇章。它引入了任务栏、开始菜单和回收站，这些设计一直沿用至今。


## Windows Phone 的失败

从 WP7 到 WP8 的内核推倒重来，导致老用户无法升级。这种内部官僚主义和对开发者生态的傲慢，最终葬送了移动端的未来。

## 永远的包袱

Windows 最强大的力量，也是它最沉重的负担，就是 向后兼容性。
为了确保十几年前的企业软件依然能在最新的 Windows 11 上运行，开发者们不得不保留了大量的陈年代码。
注册表：这个庞大且复杂的数据库，至今仍存储着从 90 年代延续至今的各种设置。
路径限制：由于 DOS 时代的遗产，很多系统路径依然存在 260 个字符的限制。
