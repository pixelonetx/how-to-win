# 注册表

Windows 的物理存储逻辑与逻辑卷管理。


## 磁盘管理与分区逻辑

在 Windows 系统中，磁盘管理不仅涉及空间的分配，还关系到系统的启动引导方式以及数据的安全性。

## 物理磁盘与分区表

在操作系统识别文件系统之前，必须先确定磁盘的分区表格式。目前主流存在两种标准：

MBR (Master Boot Record)：较旧的标准。最多支持 4 个主分区，且无法识别大于 2TB 的磁盘空间。
GPT (GUID Partition Table)：现代标准，与 UEFI 固件配合使用。支持超过 128 个分区，支持突破 2TB 限制的超大容量磁盘，且在分区表损坏时具有更强的自修复能力。



## 逻辑分区：为什么区分 C/D 盘

Windows 沿用了 DOS 时代的盘符命名传统（A、B 盘曾预留给软盘驱动器）。

C 盘：系统卷 (System Volume)
C 盘通常作为引导分区和系统根目录。它存放着 Windows 内核、驱动程序以及关键的系统组件。
Users 文件夹：存放用户文档、桌面文件及 AppData（应用配置）。
Program Files：存放 64 位应用程序。
Program Files (x86)：存放 32 位应用程序。

D 盘及其他：数据卷 (Data Volume)
将个人数据、大型游戏或项目文件存放在非系统盘是 Windows 使用中的最佳实践。
系统重装隔离：当 C 盘因系统崩溃需要格式化重装时，D 盘的数据可以完整保留。
性能考量：在机械硬盘时代，将系统与数据分开放置在不同的物理区域可以减少磁头寻道压力；在 SSD 时代，这更多是为了逻辑上的组织清晰。

## 文件系统格式

Windows 主要支持以下三种文件系统：

| 格式 | 描述 | 限制 |
| :--- | :--- | :--- |
| NTFS | Windows 默认格式。支持权限控制、压缩、加密和大于 4GB 的单文件。 | 在 Linux/macOS 上原生写入支持较复杂。 |
| exFAT | 为了闪存设备（U 盘、SD 卡）设计。跨平台兼容性极佳。 | 缺乏日志记录功能，异常拔出更容易导致文件损坏。 |
| FAT32 | 极老旧格式。主要用于引导分区（EFI 分区）或小型存储卡。 | 单个文件不能超过 4GB。 |



## EFI 系统分区与恢复分区

当你打开“磁盘管理”工具（`diskmgmt.msc`）时，你会发现一些没有盘符的隐藏分区：

EFI 系统分区 (ESP)：包含引导加载程序。系统启动时，UEFI 固件会首先读取此分区中的 `.efi` 文件。
Recovery 分区：存放 Windows 恢复环境 (WinRE)，用于系统修复或重置。

## 动态磁盘与存储池 (Storage Spaces)

除了基础的分区模式，Windows 还提供了高级存储管理功能：
跨区卷：将多个物理磁盘的空间合并为一个逻辑盘符。
镜像卷 (RAID 1)：在两个磁盘上存储相同的数据副本以防硬件故障。

## 常用管理工具

1. 磁盘管理 (GUI)：通过 `Win + X` 菜单进入，适合基础的分区调整和盘符修改。
2. Diskpart (CLI)：功能极其强大的命令行工具，可用于强制擦除分区、修改分区 ID 等高级操作。
3. PowerShell Storage Module：用于自动化管理磁盘和虚拟磁盘。
