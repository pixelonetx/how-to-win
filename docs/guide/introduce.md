# 写在前面

## 为什么要写这个

很多开发教程默认读者使用 Linux 或 macOS；很多 Windows 教程又停留在“如何重装系统”“如何关闭广告”的层面。现实是，大量学生、工程师和创作者的主力设备就是 Windows 笔记本或台式机。它可以很好地完成开发工作，但前提是你要理解它的几层结构：

1. Windows 作为桌面操作系统的使用习惯。
2. NT 内核、注册表、驱动、权限和路径等历史遗产。
3. PowerShell、包管理器、WSL、Visual Studio、VS Code 等现代工具链。
4. 不同语言和平台在 Windows 上的构建、调试、发布方式。

本书希望回答的问题不是“Windows 能不能开发”，而是“如何把 Windows 配置成一个可靠、可维护、可恢复的开发环境”。
